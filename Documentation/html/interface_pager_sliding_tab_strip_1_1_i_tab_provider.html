Detailed Description

Interface for an adapter that wants complete control over the views that are created for each of its tabs.

Note that when an Adapter supports this interface, the tab strip will still control the padding and width of each of your tabs

Called to get the view for a particular tab, either by creating a view or recycling an old one. Note - you do not have to handle the padding, background, focusability or click event for your view - this is handled by the tab strip itself through a container control.

Parameters

rootThe view that will be the root for the view you pass back - typically a FrameLayout.
positionThe position of the tab to be created.
recycledA previous view that could possibly be recycled. To indicate that this view can simply be re-bound, just return this view.
Returns
A view for the tab. MUST NOT RETURN NULL.

void PagerSlidingTabStrip.ITabProvider.RequestTabUpdate ( int  position,
string  hint = null 
)

Call this to trigger a particular tab to be updated in response to a data change.

The correct way to implement this would simply be to raise the TabUpdateRequired event, which should then be handled by the PagerSlidingTabStrip, which in turn will find the view to be updated and then call UpdateTab.

Parameters
positionThe position of the tab to be updated.
hintA hint string to be passed eventually to UpdateTab

Event Documentation

EventHandler<TabUpdateEventArgs> PagerSlidingTabStrip.ITabProvider.TabUpdated

Fired whenever a tab is updated in any way that could affect it's display. E.g., text might have changed, or a progress bar might be visible. Etc.

An implementation that surfaces this event correctly will typically be firing it in its implementation of UpdateTab.

EventHandler<TabUpdateEventArgs> PagerSlidingTabStrip.ITabProvider.TabUpdateRequired

Raised by the tab provider when it is aware of an underlying data change that will require a tab being updated through a call to UpdateTab. A PagerSlidingTabStrip instance will subscribe to this event, find the correct view for the tab in it's view hierarchy, and then pass it to UpdateTab.
